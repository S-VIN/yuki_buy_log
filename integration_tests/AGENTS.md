В папке integration_tests находятся тесты на питоне.
- Запускать интеграционные тесты надо БЕЗ докера. Запуск тестов должен быть без изоляции.
- Тесты написаны при помощи pytest
- Их надо запускать только после того, как скомпилировал и запустил сервер и очистил БД через shema.sql
- Тесты запускаются только после активации .venv
- Тесты надо писать без копипасты, сначала в комментарии пишем, какой тест-кейс тестирует тест и только потом сам тест.

Правила:
- Не используй многострочные комментарии в коде на python
- Не используй " для строк на python. Везде, где можно, используй '
- Каждый тест должен называться так, чтобы по его названию было понятно, что тестируется
- Добавляй комментарии над тестом, где пишешь тестовый сценарий
- Разбивай тесты на группы. Каждая группа тестов в своём файле
- Пиши тесты по одному, проверяя после каждого, что он работает
- Старайся в каждом тесте использовать разных user (пользовательские учетки). Создать уникального пользователя можно через req.get_new_user()

В utils находятся хелперы - классы помошники для тестов. 
- хелперы передаются в тестовые функции через фикстуры
- request_manager отвечает за get и post запросы к серверу. Также через req.get_new_user() можно создать нового пользователя.
- db_manager отвечает за подключение к БД. Через него можно делать запросы к БД, чтобы проверить, что данные добавились.
